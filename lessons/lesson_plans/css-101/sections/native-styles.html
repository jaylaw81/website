Native Styles

Before developers were allowed to control HTML pages through Cascading Style Sheets, browsers had some way of displaying HTML content visually that means it added styling (naturally) on some elements. This is both useful and annoying in different situations.

When you write CSS, one of the things you override those default browser styles and replace them with how you want the page to look, choose your own fonts, what size your text should display etc.

One way is to simply check the page as you develop and if anything seems wrong then you can correct it during -- let's say a heading wasn't positioned where you thought you told it to for example, it may be a good idea to set margin: 0; to see if the default margin was affecting with your positioning, and if this was causing the problem then you can tweak the margin from there. once you've cleared out browser styles, you can start styling same<./p>

NORMALIZE:

Some people, however, go all out and completely remove any default styles that the browser might have so they have complete control, this is usually achieved using something called a "Reset Stylesheet". This neutralizes (or normalizes) browser defaults and sets consistent style across. THen you have a clean and sensible default slate to work from.

What is Box Model?

Box Model
All HTML elements on the page is like a rectangular box.The CSS box model is essentially a box that wraps around HTML elements, and it consists of: width, height, padding, borders, margin, and the actual content.

If you don't give it a width (and that box is a block level element), then the width will remain 100% full. If padding or borders are not given, they are either browser default value or zero (likely if you are using a css reset).
